# 视觉系统与语言风格（完整规范）

布局已固化在 `assets/style.css` 与 `scripts/build_cards.py` 里，写卡片时**不需要重复这些数值**——只负责文案。这里留作核对与微调时参考。

## 画布与导出
- 每张卡片 1080×1440（小红书 3:4 竖图），导出 2 倍分辨率 2160×2880
- 卡片内边距：上下 90px、左右 85px；页面背景 #2a2a2a，卡片间距 40px

## 配色（只用这几个色号）
- 背景深炭黑 `#1D1D1F` · 薄荷绿（强调）`#5CC8A4` · 米色（正文）`#F4EFE9` · 米色暗调（次要正文）`#BFB8AC` · 分隔线灰 `#2e2e30`

## 字体
- 全部无衬线：`'Helvetica Neue','PingFang SC','Hiragino Sans GB','Microsoft YaHei',sans-serif`，不用衬线/花体

## 每张卡片的固定元素（builder 自动加，无需手写）
- 顶部 6px 薄荷绿横条
- 页眉：左「WoMen AI Lab」（AI 两字母薄荷绿），右栏目名+日期（日期薄荷绿加粗）
- 页眉下 1px 灰分隔线
- 页脚：左薄荷绿小圆点+「WoMen AI Lab」，右页码（01 / 06，自动计算）
- 右下角超大半透明数字/字水印（380px、薄荷绿、不透明度 0.06）——由每张卡的 `watermark` 字段提供

## 语言风格（非常重要，严格遵守）
这是女性写给女性看的内容。

- **不要用引号 "" 来强调**——靠断句或 `<g>薄荷绿高亮</g>`，不靠引号
- **不要用破折号 ——**
- **不要用「不是 A，而是 B」这种对仗反转句式**，读起来有 AI 腔
- 用「我们 / 我」这样的第一人称，像同伴并排说话，不要居高临下地「告诉」读者
- 句子可以短，可以有真实情绪和具体生活场景，不要堆砌行业术语
- 高亮要克制：每张卡片挑最关键的一句或一个词标绿即可，不要满屏绿

> 注意：被访者自己的回答里如果出现引号或破折号（如直接引语），在改写成卡片时尽量转成断句或高亮；只有当原话作为「真实的人声」直接引用、且确有力量时才保留。

## 内容映射（问卷字段 → 卡片）
问卷字段是天然的叙事骨架：

| 问卷字段 | 卡片用途 |
|---|---|
| What Are You Building | 「我做的是什么」——产品/功能清单页 |
| Why It Matters | 缘起、动机、为什么重要（常做封面钩子+第二张） |
| Progress Since Last Session | 进展页，挑一个最具体、最有画面感的细节 |
| One Thing Learned | 心得页，提炼一句金句 |
| Support Needed | 可融入尾页「我在找什么样的同伴」或单独一页 |
| Anything Else | 常是最有情绪的一句，适合做封面副标题或金句 |

## 一组卡片的结构
首页 1 张 + 中间页 N 张 + 尾页 1 张。默认 4–6 张（即「几页」）。

- **首页（cover）**：第一人称钩子。小标签 + 巨大标题（关键词标绿）+ 副标题 + 「→ 滑动查看」
- **中间页（middle）**：进度标记（自动）+ 大标题（关键词绿）+ 正文，可选第二段（暗调）、可选要点列表、可选小栏目（短绿线 + 小标题 + 加粗正文，一句标绿）
- **尾页（end）**：Who I am（发布者自我介绍，含名字/身份/项目）+ Who We Are（固定社群文案，builder 自动注入）+ 一行行动引导

## 发布授权（Content Permission）→ 怎么署名
- `Publish with name + social handle` → 用真名 + 小红书 handle
- `Publish with my name` → 用名字，不放 handle
- `Publish anonymously` → 匿名，尾页自我介绍用「一位社群成员」，不放名字/handle，水印别用人名相关字
- `Community only` → **不要做对外卡片**，跳过这个人

## handle 映射
问卷里的 Social Handle 字段不一定是真实小红书 ID。维护一份 `姓名 → 真实小红书 ID` 的映射放在**本地**（不要提交进公开仓库），做尾页前与用户确认。
